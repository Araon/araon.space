Accessing your local service should be simple. Trok is a lightweight tunneling tool that exposes your local services to the internet - think ngrok, but simpler.

<Image width={600} height={400} src="/projects/trok/image.png" caption="Trok - Simple TCP/HTTP Tunneling"/>

### Why Trok?

When you're developing locally and need to:
- Share a work-in-progress with a client
- Test webhooks from external services
- Debug mobile apps connecting to your local backend
- Demo something quickly without deploying

Trok makes it dead simple with a single command.

### Features

- ✅ TCP Tunneling
- ✅ HTTP Tunneling
- 🔜 UDP Tunneling (planned)
- 🔜 HTTPS Tunneling (planned)

### Installation

```sh
curl -fsSL https://trok.cloud/install.sh | sh
```

Or if you're using Nix:

```nix
# Quick test
nix run github:tuxdotrs/trok

# Add to your flake
trok = {
  url = "github:tuxdotrs/trok";
  inputs.nixpkgs.follows = "nixpkgs";
};
```

### Usage

Start a TCP tunnel:

```sh
trok tcp 3000
```

This exposes your local service running on port `3000` to a public endpoint via the trok server.

For self-hosted servers:

```sh
trok tcp 3000 -s SERVER_URL
```

### Self-Hosting

You can host your own trok server with NixOS for complete control over your tunneling infrastructure.

Built with Go for performance and simplicity. The entire client is a single binary with zero dependencies.
